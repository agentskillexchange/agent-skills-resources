# Case Studies

These case studies show how representative Agent Skill Exchange skills can fit
together in real workflows. They are not a replacement for the canonical
[agentskillexchange/skills](https://github.com/agentskillexchange/skills)
catalog.

Use them to answer three practical questions:

- Which skills might belong together in a workflow?
- What setup, permissions, and verification should a team review first?
- Which framework or resource pages explain the surrounding ecosystem?

## Workflow Cases

| Case study | What it covers |
|---|---|
| [Coding Agent Review Loop](coding-agent-review-loop.md) | PR review, implementation planning, and review-comment follow-through |
| [Security Approval Workflow](security-approval-workflow.md) | Human approval gates, dependency scanning, red-team checks, and sandboxing |
| [SRE Incident Response](sre-incident-response.md) | Kubernetes/cloud signal triage, logs, remediation drafts, and alert validation |
| [Data Research Workflow](data-research-workflow.md) | Source-backed research plus database and SQL inspection patterns |
| [Runtime Ops: OpenClaw](runtime-ops-openclaw.md) | Day-2 runtime operations, backup, hardening, and workspace continuity |

## How To Use These

Start with a scenario close to your own work, then inspect the referenced ASE
skills in the canonical catalog. Before adoption, check source provenance,
install steps, permission boundaries, and verification guidance.
