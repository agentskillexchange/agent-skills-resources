# Maintenance And Freshness

Agent skill resources age quickly. Tool commands change, docs move, APIs add
permissions, and runtimes introduce new safety models.

## Freshness Checklist

- Links still resolve.
- Official/community labels still match ownership.
- Install commands still work.
- Required permissions are still accurate.
- Screenshots or examples are not stale.
- Popularity claims have dates or are removed.
- Known risks still reflect current behavior.

## Review Cadence

| Resource type | Suggested review |
|---|---|
| Official docs | Every 60-90 days |
| Security guidance | Every 30-60 days |
| Community lists | Every 90 days |
| Install commands | When package managers or runtimes change |
| Team playbooks | After every pilot or incident |

## Remove vs Relabel

Remove a resource when it is dead, misleading, or no longer relevant. Relabel it
when ownership changes or when a community resource becomes official.

## Maintenance Evidence

Record what changed, why it changed, and what validation was run. Short notes
are better than silent churn.
