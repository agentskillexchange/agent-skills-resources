# Agno

Agno is an agent framework for building agent applications with models, tools,
knowledge, memory, and multi-agent patterns.

## Where It Fits

Agno fits the lightweight agent application layer. It is relevant when teams want
to build practical agent apps without starting from a large orchestration stack.

## Skill-Like Concepts

- agents
- tools
- knowledge and memory
- multi-agent coordination
- model integrations
- application-level monitoring and policy

## Best Fit

- lightweight agent apps
- tool-using assistants
- workflows that combine knowledge, memory, and model calls
- teams comparing multiple agent frameworks quickly

## When Not To Use It

- workflows that need a mature enterprise governance layer from day one
- simple one-command automations
- catalog browsing or static skill discovery

## Safety And Rollout

- Define tool scopes and memory boundaries.
- Test knowledge retrieval before relying on output.
- Keep pilot workflows narrow.
- Require review before write actions or external communication.
