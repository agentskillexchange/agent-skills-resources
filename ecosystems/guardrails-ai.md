# Guardrails AI

Guardrails AI provides validation and guardrail patterns for LLM application
inputs and outputs.

## Where It Fits

Guardrails AI fits the runtime validation layer. It is useful when a workflow
needs structured checks before trusting or releasing model output.

## Skill-Adjacent Concepts

- input validation
- output validation
- structured checks
- validators
- runtime guardrails
- failure handling

## Best Fit

- structured outputs
- policy-sensitive responses
- workflows that need validation before tool use or publication

## When Not To Use It

- workflows where policy is undefined
- cases where human approval is still required but missing
- security review without runtime evidence
