# Popular Agent Ecosystems

Popular agent ecosystems usually expose one or more reusable surfaces: agents,
tools, workflows, memory, evals, guardrails, hooks, and MCP integrations. In
this repo, they are covered as skill-adjacent systems: places where reusable
agent instructions and workflow patterns can live.

## How To Read This Section

- Use framework pages for runtime-specific skill surfaces.
- Use ecosystem pages for broader agent frameworks and SDKs.
- Use the cookbook when you want design and review patterns.
- Use checklists when you need a short adoption artifact.

## Ecosystems Covered

| Ecosystem | Why it matters for skills |
|---|---|
| CrewAI | Encodes reusable roles, tasks, crews, flows, tools, memory, and guardrails. |
| AutoGen | Focuses on multi-agent patterns, agent runtimes, tools, extensions, and Studio workflows. |
| Semantic Kernel | Connects agents, plugins, planners, memory, and enterprise app integration. |
| LlamaIndex | Bridges data, tools, RAG, agents, workflows, evals, and observability. |
| Pydantic AI | Gives Python teams typed agents, toolsets, MCP, evals, durable workflows, and approval patterns. |
| Haystack | Provides production RAG and agent pipelines with tools, evaluators, and MCP serving. |
| Strands Agents | Provides SDK-level agent patterns, tools, hooks, observability, and coding-agent setup. |
| Agno | Supports agent apps with tools, memory, knowledge, models, and multi-agent coordination. |
