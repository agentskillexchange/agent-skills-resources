# Builder Track

## Who It Is For

Skill authors, automation owners, and developers turning repeated agent work
into reusable instructions.

## Expected Time

60-90 minutes.

## Prerequisites

- You can name a repeated workflow.
- You know the tools, APIs, repos, or documents the workflow touches.
- You can run or observe the workflow manually at least once.

## Practice Exercise

Draft a one-page skill outline:

- workflow name
- source URL
- prerequisites
- steps
- verification command or evidence
- permissions needed
- failure mode and rollback note

## What Good Looks Like

- The skill solves a narrow repeated workflow.
- It cites the source it depends on.
- Setup is specific enough for a first run.
- Verification is observable.
- Permissions and risky operations are named.

## Common Mistakes

- Writing broad instructions that cannot be tested.
- Claiming multi-framework support without naming differences.
- Copying upstream docs without explaining the agent workflow.
