# Agent Skills Curriculum

Use this curriculum to move from concepts to reviewable team adoption. Pick the
track that matches your current role; you do not need to read everything first.

| Track | For | Outcome |
|---|---|---|
| [Beginner](beginner-track.md) | Developers and operators new to agent skills | Understand skills, tools, MCP, agents, and workflows |
| [Builder](builder-track.md) | Skill authors and automation owners | Design source-backed, testable skills |
| [Evaluator](evaluator-track.md) | Reviewers, security partners, and maintainers | Assess quality, safety, permissions, and evidence |
| [Team Lead](team-lead-track.md) | Engineering, platform, and ops leads | Pilot, monitor, and roll out skills with guardrails |

## Suggested Order

1. Read the Beginner track to normalize vocabulary.
2. Use the Builder track when creating or adapting a skill.
3. Use the Evaluator track before a pilot or public recommendation.
4. Use the Team Lead track when a workflow needs adoption planning.

## Evidence Standard

Every track uses the same basic evidence standard:

- source URL is clear
- setup and permissions are named
- workflow fit is specific
- verification result is repeatable
- risks and approval points are visible
